Class: Sprite

Sprite(props)

new Sprite(props)

Base element for displaying animations.

Animation is provided through an array of frames. Each frame will be shown for a specified duration; after that duration, the next frame is shown. Animation can be looped, or terminated.

Animation lifetime of a Sprite can be controlled through the parameters, ttl (Time-to-live), loop (boolean, loop or not), and Loops-to-Live. If any of these parameters causes the Sprite's animation to stop, an event, EventType.SPRITE_ANIMATION_DONE, will be emitted.

Note that if the animation stops, the sprite itself will remain on screen; if no longer needed, it must be explicitly hidden or removed.

This is an abstract class; you must provide an implementation that overrides renderFrame(), and AnimationFrames that describe what should be rendered.

nudge(offsetX, offsetY, decay, interval, intervalDecay, callback)

Nudge the element, make it move back and forth rapidly. Provides options for decays of movement range (will make movement less extreme over time), and of time interval (movement becomes more rapid over time). Automatically returns to its starting position upon completion. If the offsets or the interval time reaches zero due to the decay parameters, the nudge will finish. Will throw an error if the parameters will cause the nudge to repeat for >= 1000 intervals.
Parameters:
Name Type Description
offsetX number The maximum (positive and negative) horizontal offset to move the element.
offsetY number The maximum (positive and negative) vertical offset to move the element.
decay number After hitting the current offset and reversing direction, reduce the offsets by this amount.
interval number How rapidly (milliseconds) the element should move to the offset, before reversing direction.
intervalDecay number Reduces the interval time after each iteration.
callback function A function to call when the nudge has completed.
